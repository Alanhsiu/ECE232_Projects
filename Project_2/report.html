Q21: Meaning of Homogeneity and Completeness

Homogeneity (hhh) measures whether each detected community consists of members that all belong to the same circle. Formally:

h=1−H(C∣K)H(C)h = 1 - \frac{H(C \mid K)}{H(C)}h=1−H(C)H(C∣K)​

H(C∣K)H(C \mid K)H(C∣K) is the residual uncertainty about which circle a node belongs to, given that we already know its community assignment. If h=1h = 1h=1, every community is "pure" — all its members come from a single circle. If h=0h = 0h=0, knowing a node's community tells us nothing about its circle. A negative hhh would mean communities are even more mixed than the overall circle distribution, i.e., the algorithm actively mixes circles together rather than keeping them separate.

Completeness (ccc) measures whether all members of the same circle are assigned to the same community. Formally:

c=1−H(K∣C)H(K)c = 1 - \frac{H(K \mid C)}{H(K)}c=1−H(K)H(K∣C)​

H(K∣C)H(K \mid C)H(K∣C) is the residual uncertainty about which community a node belongs to, given its circle. If c=1c = 1c=1, every circle is fully captured by a single community — no circle is split across multiple communities. If c=0c = 0c=0, knowing a node's circle tells us nothing about which community it lands in. A negative ccc would mean circles are more fragmented across communities than a random assignment would produce.

Together, hhh and ccc are analogous to precision and recall in information retrieval: homogeneity reflects the internal purity of detected communities (each community should not mix different circles), while completeness reflects how well the algorithm keeps each circle intact (all members of a circle should end up in the same community).

Q24: Node2Vec + Classifier

How Node2Vec Finds Node Features

Node2Vec learns low-dimensional node embeddings by simulating biased random walks on the graph and training a skip-gram model (Word2Vec) on the resulting walk sequences. At each step of a walk currently at node vvv (having arrived from ttt), the transition probability to the next node xxx is:

πvx=αpq(t,x)⋅wvx\pi_{vx} = \alpha_{pq}(t, x) \cdot w_{vx}πvx​=αpq​(t,x)⋅wvx​

where αpq\alpha_{pq}αpq​ depends on the distance dtxd_{tx}dtx​ between xxx and the previous node ttt:

αpq(t,x)={1/pif dtx=0 (return to t)1if dtx=1 (stay local)1/qif dtx=2 (explore further)\alpha_{pq}(t, x) = \begin{cases} 1/p & \text{if } d_{tx} = 0 \text{ (return to } t\text{)} \\ 1 & \text{if } d_{tx} = 1 \text{ (stay local)} \\ 1/q & \text{if } d_{tx} = 2 \text{ (explore further)} \end{cases}αpq​(t,x)=⎩⎨⎧​1/p11/q​if dtx​=0 (return to t)if dtx​=1 (stay local)if dtx​=2 (explore further)​

  • Return parameter ppp: small ppp encourages revisiting recent nodes (DFS-like, captures structural roles).
  • In-out parameter qqq: small qqq favours local exploration (BFS-like, captures community membership).

In our implementation we use p=q=1p = q = 1p=q=1 (unbiased DeepWalk-style walk), embedding dimension 128, walk length 80, 10 walks per node, and window size 10. The skip-gram model predicts surrounding nodes in a walk, so nodes that frequently co-occur in walks receive similar embedding vectors.

Why Does Node2Vec Outperform Text Features?

The key reason is the extreme mismatch between training set size and feature dimensionality. With only 140 labelled nodes and 1433-dimensional sparse binary features, all three classifiers suffer from the curse of dimensionality:

  • MLP with 1433-dimensional input and only 140 training samples is severely under-constrained, achieving only 35% accuracy — barely above a naive majority-class baseline. This is a textbook case of overfitting due to insufficient data.
  • SVM cannot reliably estimate a large-margin decision boundary in 1433 dimensions from 140 examples, falling to 45.4%.
  • Random Forest is more robust to high dimensions but still limited to 57.5% because the 1433-dimensional binary bag-of-words vectors are extremely sparse, providing weak signal per feature.

Node2Vec embeddings (128-d, dense, continuous) are far better conditioned for small-sample learning. The random walks compress graph structure into a compact representation where citation proximity — a strong proxy for topic similarity — is encoded geometrically, providing an inductive bias well-aligned with the classification task. All 2708 nodes participate in walk generation regardless of label availability, so the 128-d embeddings capture global graph structure using the full graph, not just the 140 labelled nodes.

Best Classification Accuracy

This confirms that in the low-data regime, the high-dimensional sparse text features add more noise than discriminative signal, diluting the informative 128-d Node2Vec embedding and hurting the classifier.

Q25: Personalized PageRank

Method

We run personalized PageRank as a biased random walk on the Giant Connected Component (GCC) of the Cora citation graph (2,485 nodes, 5,069 edges after converting to undirected). 18 of the 140 training seed nodes fall outside the GCC, leaving 122 GCC seed nodes. Similarly, 85 of the 1,000 test nodes are outside the GCC; we evaluate on the remaining 915 GCC test nodes. For each of the 7 classes, we launch 1,000 random walks from each seed node of that class (122,000 total walks). Each walk has length 100 steps.

At every step from the current node x0x_0x0​ with neighbours x1,…,xkx_1, \ldots, x_kx1​,…,xk​:

  • With probability α\alphaα: teleport to a uniformly random seed node of the current class.
  • With probability 1−α1 - \alpha1−α: transition to neighbour xix_ixi​ with probability proportional to the softmax of dot-product similarity between text features:

pi=exp⁡(x0⋅xi)∑j=1kexp⁡(x0⋅xj)p_i = \frac{\exp(x_0 \cdot x_i)}{\sum_{j=1}^{k} \exp(x_0 \cdot x_j)}pi​=∑j=1k​exp(x0​⋅xj​)exp(x0​⋅xi​)​

We maintain a class-wise visit counter for every node. The predicted label for an unlabelled node is the class whose walks visited it most often. Nodes that receive zero visits across all classes are assigned the majority class of the GCC training set. We vary α∈{0,0.1,0.2}\alpha \in \{0, 0.1, 0.2\}α∈{0,0.1,0.2}.
